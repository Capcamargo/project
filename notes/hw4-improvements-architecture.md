# HW4 — Улучшения MVP и архитектура

## Что нужно улучшить в MVP
- сократить путь до первого результата;
- сделать форму подбора короче и понятнее;
- показать причины, почему рекомендация подходит;
- убрать лишние шаги между подбором и просмотром результата.

## Архитектура MVP
### Frontend / mobile
TypeScript + React Native + Expo

### Backend / data
Supabase

## Почему это подходит проекту
Такой стек позволяет быстро собрать первую версию продукта и сохранить возможность дальнейшего роста в сторону iOS и Android без разделения кодовой базы.

## Базовые блоки системы
1. Экран лендинга.
2. Форма подбора подарка.
3. Экран рекомендаций.
4. Хранение запросов пользователя.
5. Хранение результатов рекомендаций.
