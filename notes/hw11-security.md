# HW11 — Безопасность MVP

## Основные риски
- доступ к чужим данным;
- изменение пользовательских записей в обход логики приложения;
- хранение чувствительных ключей в клиенте;
- доверие только фронтенду без ограничений на уровне базы.

## Что важно для GiftMatch
1. Пользователь должен видеть только свои анкеты и свои рекомендации.
2. Все ограничения по данным должны быть продублированы на уровне базы через RLS.
3. В клиент нельзя помещать чувствительные серверные ключи.
4. Монетизация и чувствительная логика не должны проверяться только на фронте.

## Вывод
Даже учебный MVP должен быть устроен так, чтобы пользователь не мог свободно читать или менять чужие данные.
