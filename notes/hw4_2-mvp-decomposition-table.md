# HW4.2 — Декомпозиция MVP

## Основные блоки MVP GiftMatch

| № | Блок | Что делает |
|---|------|-------------|
| 1 | Лендинг | Объясняет ценность сервиса и ведет к началу подбора |
| 2 | CTA-кнопка | Переводит пользователя к форме |
| 3 | Форма подбора | Собирает повод, бюджет и интересы |
| 4 | Проверка формы | Не дает отправить пустые данные |
| 5 | Логика подбора | Формирует список рекомендаций |
| 6 | Экран результатов | Показывает подходящие идеи подарков |
| 7 | Карточка подарка | Дает название, краткое объяснение и ценовой ориентир |
| 8 | Повторный подбор | Позволяет изменить параметры и обновить результат |
| 9 | Регистрация | Создает пользовательский аккаунт |
| 10 | Вход | Возвращает пользователя к его данным |
| 11 | Профиль | Хранит базовые сведения о пользователе |
| 12 | Хранение запросов | Сохраняет анкеты пользователя |
| 13 | Хранение рекомендаций | Сохраняет результаты подбора |
| 14 | RLS-ограничения | Закрывают доступ к чужим данным |
| 15 | Аналитика | Позволяет понять, где пользователь теряется |
| 16 | Публикация MVP | Делает продукт доступным по публичной ссылке |

## Вывод
GiftMatch можно разложить на понятные функциональные блоки, и это помогает не распыляться на второстепенные возможности раньше времени.
