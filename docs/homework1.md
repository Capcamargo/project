# Homework 1 — GiftMatch

## Название проекта
GiftMatch — сервис персонализированного подбора подарков.

## Проблема
Выбор подарка часто занимает слишком много времени и вызывает стресс. Пользователю сложно придумать что-то подходящее, небанальное и уместное по бюджету. В результате поиск растягивается на множество сайтов и маркетплейсов, а уверенности в выборе все равно нет.

## Целевая аудитория
Основная аудитория MVP — занятые пользователи 18–35 лет, которым нужно быстро подобрать подарок другу, партнеру, родственнику или коллеге. Это люди, которые ценят скорость, удобство и понятный результат.

## Ценность будущего MVP
GiftMatch помогает пользователю за несколько минут получить персонализированные идеи подарков по поводу, интересам и бюджету. Ценность продукта заключается в экономии времени, снижении стресса и более уверенном выборе.

## Основной JTBD
Когда мне нужно выбрать подарок близкому человеку и я не уверен, что ему подойдет, я хочу за несколько минут получить уместные идеи с учетом интересов, повода и бюджета, чтобы быстро выбрать хороший подарок без стресса и долгого поиска.

## Ключевые функции MVP
1. Форма подбора подарка.
2. Ввод повода, бюджета и интересов получателя.
3. Персонализированная подборка рекомендаций.
4. Карточки подарков с кратким описанием.
5. Переход к дальнейшему действию.

## Основной пользовательский сценарий
1. Пользователь заходит на страницу сервиса.
2. Заполняет короткую форму.
3. Получает персонализированную подборку.
4. Сравнивает варианты.
5. Переходит к просмотру или покупке.

## Выбранный стек
### Frontend / mobile
TypeScript + React Native + Expo

### Backend / data
Supabase

## Почему выбран именно этот стек
Этот стек подходит для вайб-кодинга MVP, потому что позволяет не распыляться на несколько отдельных платформ. React Native и Expo дают основу для дальнейшего развития под iOS и Android, а TypeScript делает кодовую базу более предсказуемой и удобной для роста. Supabase закрывает базовые backend-задачи и позволяет сфокусироваться именно на продукте и пользовательском сценарии.
