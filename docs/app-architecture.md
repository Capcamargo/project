# Архитектура приложения GiftMatch

## Общая логика
GiftMatch строится вокруг одного основного пользовательского сценария: пользователь вводит параметры подбора подарка, получает рекомендации и при желании сохраняет результаты.

## Слои приложения

### 1. Клиентский слой
Клиентская часть отвечает за интерфейс, форму подбора, экран результатов, регистрацию, вход и отображение пользовательских данных.

### 2. Слой данных
Supabase используется для хранения профилей пользователей, анкет подбора и сохраненных рекомендаций.

### 3. Логика рекомендаций
На раннем этапе рекомендации могут быть моковыми или формироваться по простой логике. Позже этот слой можно заменить на более умную модель подбора.

## Основные сущности
- профиль пользователя;
- запрос на подбор подарка;
- рекомендация подарка.

## Главный принцип
Архитектура не должна быть сложнее самого MVP. Все решения подчинены одной задаче: как можно быстрее довести пользователя до полезного результата.
