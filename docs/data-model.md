# Модель данных GiftMatch

## Основные сущности

### 1. Profile
Хранит базовую информацию о пользователе: идентификатор, email, отображаемое имя, даты создания и обновления.

### 2. Gift request
Описывает один запрос на подбор подарка. Включает повод, бюджет, интересы получателя, тип отношений и дополнительные заметки.

### 3. Gift recommendation
Описывает конкретную рекомендацию, связанную с запросом пользователя. Может включать название, краткое описание, категорию, ценовой ориентир и ссылку.

## Связи между сущностями
- один пользователь может создать много запросов;
- один запрос может иметь много рекомендаций;
- каждая рекомендация относится к одному пользователю и одному запросу.

## Почему модель именно такая
Для MVP важно хранить не весь каталог рынка подарков, а действия пользователя и результаты подбора. Это позволяет быстро построить первый полезный сценарий и не усложнять систему раньше времени.
